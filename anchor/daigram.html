1
Seller signs & submits transaction
Seller provides price (lamports) and expiry (unix timestamp) as instruction arguments. Seller pays rent for the new Listing PDA and Escrow ATA.
price: u64 expiry: i64
↓
2
Anchor initialises Listing PDA
A new account is created at the PDA derived from [b"listing", seller, mint]. The bump, price, expiry, seller pubkey, and mint pubkey are stored on-chain.
seller mint price expiry bump
↓
3
Metaplex TransferV1 CPI called
The program invokes the Metaplex Token Metadata program via TransferV1CpiBuilder. This path handles both standard NFTs and pNFTs (Programmable NFTs with royalty enforcement).
mpl-token-metadata v4 TransferV1 amount: 1
↓
4
Metaplex validates metadata & rules
Metadata account, Master Edition, and (for pNFTs) Token Record accounts are verified. Royalty rules from the NFT's RuleSet are enforced. Sysvar Instructions is read to detect CPI depth.
Metadata PDA MasterEdition TokenRecord (pNFT) SysvarInstructions
↓
5
NFT transferred to Escrow ATA
Token moves from Seller's ATA → Escrow ATA (authority = Listing PDA). The NFT is now locked in escrow until bought or the listing is cancelled.
Seller ATA → Escrow ATA authority: seller (signer)
pNFT note: For Programmable NFTs, the seller_token_record and escrow_token_record accounts must be passed. These track delegation state and enforce rule sets (e.g. creator royalties). For standard SPL NFTs they can be omitted (None).

buy_nft instruction — buyer initiates

🛒
Buyer

💰

lamport rebalance
🔒
Escrow ATA
NFT source in escrow
📦
Buyer ATA
NFT destination
1
Buyer signs & submits transaction
Buyer submits the buy_nft instruction. Anchor validates that the Listing PDA seeds match and the account is not already closed.
↓
2
Expiry guard checked
The program reads Clock::get()?.unix_timestamp and compares it to listing.expiry. If the listing has expired, the transaction aborts with ErrorCode::ListingExpired.
Clock sysvar unix_timestamp ≤ expiry
↓
3
SOL transferred — Buyer → Seller
Lamports are moved directly by mutating both accounts' lamport fields. This is a native SOL transfer (no CPI needed). Amount = listing.price.
buyer lamports -= price seller lamports += price
↓
4
Metaplex TransferV1 CPI — PDA signed
The program calls TransferV1CpiBuilder::invoke_signed() using the Listing PDA's seeds as the signer. The PDA acts as the authority over the Escrow ATA.
seeds: [listing, seller, mint, bump] invoke_signed
↓
5
NFT transferred to Buyer's ATA
Token moves from Escrow ATA → Buyer ATA. Metaplex validates Metadata, MasterEdition, and TokenRecord accounts, then performs the SPL token transfer internally.
Escrow ATA → Buyer ATA
↓
6
Listing PDA closed — rent returned
Anchor's close = seller constraint automatically closes the Listing PDA at the end of the instruction, returning its rent-exempt lamports to the seller.
close = seller rent reclaimed
Security note: The SOL transfer uses direct lamport manipulation, which requires both buyer and seller accounts to be mut. The Listing PDA bump is stored on-chain so the program can re-derive and sign without user input, preventing seed collision attacks.

cancel_listing instruction — seller initiates

Listing PDA

1
Seller signs cancel transaction
Only the original seller can cancel. The has_one = seller constraint on the Listing PDA enforces this automatically — no separate require! check needed.
has_one = seller seeds verified
↓
2
Metaplex TransferV1 CPI — PDA signed
Same PDA-signed CPI pattern as buy_nft. The Listing PDA authorises moving the NFT out of the Escrow ATA back to the seller.
invoke_signed Listing PDA as authority
↓
3
NFT returned to Seller's ATA
Token moves from Escrow ATA → Seller ATA. Metaplex validates and updates all pNFT state (TokenRecord delegation cleared).
Escrow ATA → Seller ATA
↓
4
Listing PDA closed — rent returned
The Listing PDA is closed via close = seller, returning all rent-exempt SOL to the seller. The on-chain listing record is fully removed.
close = seller state fully cleaned up
Important: The Escrow ATA (owned by the now-closed Listing PDA) should be closed separately or via an additional cleanup instruction to reclaim its rent. Consider adding close = seller to the escrow token account after the transfer.

Complete lifecycle — all paths

A
list_nft — NFT enters escrow
Seller → Listing PDA created. NFT moves Seller ATA → Escrow ATA via Metaplex TransferV1. Listing stores price, expiry, seller, mint, bump.
SOL: seller pays rent NFT: seller → escrow
↓
B
buy_nft
Expiry checked. SOL flows Buyer → Seller. NFT flows Escrow → Buyer via signed CPI. Listing PDA closed, rent to seller.
SOL: buyer → seller NFT: escrow → buyer PDA closed
C
cancel_listing
Seller-only. NFT flows Escrow → Seller ATA via signed CPI. Listing PDA closed, rent to seller.
NFT: escrow → seller PDA closed
✓
D
Final state — listing fully resolved
In both paths the Listing PDA is closed and all on-chain state is cleaned up. The NFT is held by exactly one party (buyer or seller) and the escrow ATA is empty.
Listing PDA: closed Escrow ATA: 0 balance Rent reclaimed
Metaplex CPI chain: Your program → Token Metadata Program → SPL Token / Token-2022. The sysvar_instructions account is passed so Metaplex can detect nested CPI depth, which is used to enforce pNFT rules and prevent certain delegate/authority exploits. Token record accounts must be passed for pNFTs and can be None for standard NFTs — the same instruction handles both cases cleanly.
